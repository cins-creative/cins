Bạn mở link trên điện thoại, vuốt qua màn hình đăng nhập app giao hàng, bấm «Theo dõi đơn» và thấy chuyển sang màn hình bản đồ với nút gọi tài xế — trước khi developer viết code, team sản phẩm thường đã dựng prototype có thể bấm thử trong Figma. Designer vẽ UI, nối màn hình bằng hotspot, gửi link cho bạn bè trong lớp «test» xem luồng có hợp lý không.

Figma là công cụ thiết kế giao diện và cộng tác UI/UX chạy trên trình duyệt (và app desktop) — tiêu chuẩn de facto tại startup Việt Nam và Big Tech toàn cầu. Học sinh THPT làm mockup app ý tưởng, dự án khởi nghiệp số hay portfolio design đều có thể bắt đầu miễn phí với gói Figma Education hoặc Starter.

01Figma là gì?

Figma là nền tảng thiết kế UI/UX và prototype trên cloud — file lưu online, nhiều người chỉnh sửa đồng thời, comment trực tiếp trên khung thiết kế. Không giống Photoshop (chỉnh ảnh pixel), Figma xây cho digital product: frame (artboard), component tái sử dụng, auto layout, design token và Dev Mode cho developer lấy spec CSS.

Giao diện gồm canvas giữa, panel layer bên trái, properties bên phải; chế độ Prototype để nối frame thành luồng tương tác. File .fig đồng bộ real-time — phù hợp nhóm học sinh THPT làm dự án nhóm: một bạn UX wireframe, một bạn UI polish, cả lớp comment mà không gửi file qua lại.

  • Frame & auto layout — khung iPhone, Android, desktop; layout co giãn như Flexbox
  • Component & variants — nút, input, card tái sử dụng; đổi state (default, hover, disabled)
  • Prototype — hotspot, smart animate, overlay modal, scroll behavior
  • Dev Mode & handoff — đo khoảng cách, copy CSS, export SVG/PNG cho developer

Vị trí trong quy trình sản xuất

Với dự án trường học, Figma thay slide tĩnh — mockup app «thư viện số lớp» có thể demo trước hội đồng. Thường làm việc cùng FigJam (brainstorm), Notion (brief) và GitHub (implementation).

02Tính năng cốt lõi

1. Thiết kế UI với frame, vector và auto layout

Figma cung cấp công cụ vector (rectangle, ellipse, pen), text với Google Fonts, và auto layout — nhóm element tự căn khoảng cách khi thêm nút hoặc đổi copy, tương tự CSS Flexbox. Bạn dựng wireframe grayscale trước, sau đó áp màu, typography và icon — quy trình «structure trước, visual sau» mà studio UX chuyên nghiệp luôn theo.

Với học sinh THPT, bài tập thực tế: thiết kế 6–10 frame app «đặt suất ăn căn tin» hoặc «tra cứu điểm thi» — dùng grid 8px, font Inter hoặc Be Vietnam Pro, màu thương hiệu trường. Auto layout giúp danh sách món ăn hoặc bảng điểm co giãn mà không phải căn tay từng pixel.

  • Frame preset — iPhone 15, Pixel 8, desktop 1440px có sẵn
  • Auto layout — padding, gap, hug/fill contents cho responsive cơ bản
  • Styles — color, text style tái sử dụng xuyên suốt file

2. Component, variants & design system nhẹ

Component là khối tái sử dụng: tạo nút Primary một lần, instance trên mọi màn hình — sửa Main Component thì tất cả instance cập nhật. Variants gom state (size S/M/L, type primary/secondary) trong một component set — giảm lặp lại và giữ UI nhất quán.

Dự án THPT có thể có «design system mini»: 3 màu brand, 2 font size, 5 component (button, input, card, navbar, tab bar). Khi giáo viên yêu cầu đổi màu chủ đạo, bạn sửa style một chỗ thay vì 40 màn hình — đây là tư duy scalable design mà intern UI tại startup được kỳ vọng.

3. Prototype, test luồng & Dev Mode handoff

Chuyển sang tab Prototype, kéo mũi tên từ nút «Đăng nhập» sang frame tiếp theo, chọn transition (dissolve, smart animate). Gửi link Present — người xem bấm thử trên trình duyệt như app thật. Thu thập comment («màn này thiếu nút quay lại») trước khi code.

Dev Mode (gói trả phí team) giúp developer đo spacing, copy mã CSS/iOS/Android, export icon @2x. Luồng điển hình: designer handoff file Figma → dev implement React/Flutter → designer review pixel trên staging. Hiểu prototype + handoff giúp bạn nói chung ngôn ngữ với lập trình viên — kỹ năng cross-functional rất giá trị.

03So sánh với phần mềm tương tự

Hai công cụ hay được đặt cạnh Figma nhất là Adobe XD (cùng hệ Adobe, Adobe đã ngừng phát triển tính năng mới) và Sketch (Mac-only, từng là tiêu chuẩn UI trước Figma). Không có câu trả lời «đúng duy nhất» — tùy team, hệ điều hành và nhu cầu cộng tác cloud.

Figma vs Adobe XD

Cả hai đều thiết kế UI và prototype cho app/web. XD tích hợp Creative Cloud, quen với designer dùng Photoshop/Illustrator; Adobe đã chuyển hướng sang Figma partnership và không còn đẩy tính năng XD mới. Figma mạnh về cộng tác real-time, plugin community, FigJam và Dev Mode — hầu hết studio Việt Nam tuyển Figma trên JD UI designer hiện nay.

Điểm mạnh và yếu so sánh

Figma mạnh hơn: cloud-first, cộng tác đa nền tảng (Windows/Mac/web), plugin & community lớn, FigJam, Dev Mode, tiêu chuẩn tuyển dụng UI hiện tại.

Adobe XD mạnh hơn: quen thuộc nếu team đã trả Creative Cloud, một số workflow với After Effects/Illustrator cũ.

Chọn Figma khi: dự án mới, làm việc nhóm, học UI/UX từ THPT đến intern — hầu hết trường hợp nên chọn Figma.

Chọn XD khi: duy trì file cũ, công ty bắt buộc stack Adobe và chưa migrate.

Figma vs Sketch

Sketch chạy chỉ trên macOS, file local (.sketch) với hệ plugin mạnh — từng là công cụ số 1 cho UI trước 2018. Figma browser-based, miễn phí tier rộng, không bắt buộc Mac. Sketch giờ có cloud collaboration nhưng Figma vẫn dẫn về số người dùng đồng thời và handoff tích hợp.

04Học Figma như thế nào?

1
Tạo tài khoản và làm quen canvas

Đăng ký Figma miễn phí (hoặc Education nếu trường hỗ trợ), tạo file Draft đầu tiên. Tuần đầu luyện zoom, pan, chọn object, frame tool (F), rectangle, text — đặt frame iPhone preset và vẽ vài hình. Mục tiêu là thao tác nhanh trên canvas trước khi học component — nhiều bạn THPT bối rối vì layer lồng nhau; hãy đặt tên layer rõ ràng ngay từ đầu.

2
Wireframe grayscale 4–6 màn hình

Chọn một app đơn giản (todo list, menu căn tin, hồ sơ CLB). Vẽ wireframe chỉ xám/trắng: header, list, nút FAB — chưa cần màu đẹp. Tập trung thông tin và thứ bậc: cái gì người dùng thấy trước, CTA ở đâu. Nhờ 2–3 bạn bạn cùng lớp nhìn 5 giây mỗi màn và trả lời «app này làm gì» — nếu không trả lời được, wireframe chưa rõ.

3
Mockup visual, auto layout & component cơ bản

Áp color style, typography (Be Vietnam Pro cho tiếng Việt), icon từ Figma Community. Tạo component Button và Input; dùng auto layout cho list item. Duplicate frame sang dark mode nếu muốn thử thách thêm. Mục tiêu: 6–8 màn hình visual nhất quán — đủ cho slide báo cáo môn Tin hoặc cuộc thi học sinh sáng tạo.

4
Prototype luồng chính và user test

Nối frame: splash → đăng nhập → home → chi tiết → quay lại. Thêm overlay cho dialog xác nhận; thử smart animate cho chuyển tab. Gửi link prototype cho 5 người (bạn, em, phụ huynh) làm task «đặt một món ăn» — ghi lại chỗ họ vấp. Iteration theo feedback là core UX, không phải «vẽ xong là xong».

5
Dự án case study hoàn chỉnh

Chọn brief có story: «App nhắc uống nước cho học sinh thi cử» hoặc «Nền tảng chia sẻ tài liệu lớp». Viết 1 trang: vấn đề, persona, giải pháp; trong Figma có wireframe, final UI, prototype link. Giai đoạn này dạy quy trình sản phẩm thật — research ngắn, ưu tiên MVP, không thiết kế 40 màn không ai dùng.

6
Portfolio và bước nghề nghiệp

Đăng case study lên Behance, personal site hoặc Notion — embed prototype Figma, mô tả vai trò và quyết định design. Tham gia Figma Community duplicate file mentor, học thêm accessibility (contrast, touch target 44px). Tìm intern UI tại startup VN thường cần Figma + hiểu UX cơ bản; chứng chỉ ít quan trọng hơn 2–3 project chất lượng trong portfolio.
