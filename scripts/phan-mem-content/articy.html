Bạn chơi game RPG và gặp NPC nói chuyện khác nhau tùy lựa chọn của bạn — «Giúp dân làng» mở nhánh nhiệm vụ A, «Bỏ qua» thì câu chuyện rẽ sang hướng khác. Phía sau màn hình, writer và designer thường không viết hết trong Word mà dựng sơ đồ cốt truyện bằng công cụ chuyên dụng như Articy Draft.

Articy Draft do Articy Software phát triển — phần mềm thiết kế narrative cho game: flowchart hội thoại, bảng nhân vật, biến trạng thái, xuất dữ liệu sang Unity/Unreal. Với học sinh THPT thích viết truyện tương tác hoặc làm game visual novel, Articy là «bản đồ» giúp bạn không lạc nhánh khi cốt truyện phình to.

01Articy Draft là gì?

Articy Draft là công cụ authoring narrative — tập trung vào cấu trúc câu chuyện, hội thoại và logic rẽ nhánh thay vì code gameplay hay dựng đồ họa. Bạn tạo flow fragment (mảnh luồng), nối các dialogue node và condition (điều kiện), gán speaker cho từng dòng thoại — toàn bộ hiển thị dạng sơ đồ trực quan trên canvas.

Khác với Google Docs hay Notion chỉ liệt kê văn bản tuần tự, Articy mô hình hóa cây quyết định và biến game (ví dụ `trust_level`, `has_key`) để programmer import vào engine. Giao diện có navigator dự án, template nhân vật, hub liên kết nhiều flow — ban đầu trông phức tạp, nhưng mỗi panel phục vụ một vai trò trong pipeline game có writing team.

  • Flowchart narrative — node hội thoại, jump, hub, condition trên canvas kéo-thả
  • Entity database — nhân vật, địa điểm, item, template thuộc tính tái sử dụng
  • Variables & conditions — logic rẽ nhánh theo trạng thái người chơi
  • Export plugin — JSON/XML sang Unity, Unreal; tích hợp Yarn/Ink qua workflow trung gian

02Tính năng cốt lõi

1. Flowchart hội thoại & rẽ nhánh

Trái tim của Articy là canvas flow: mỗi node có thể là đoạn hội thoại, narration, player choice hoặc instruction cho engine. Bạn nối cạnh (connection) giữa các node; thêm Condition để chỉ đi nhánh khi biến đạt ngưỡng — ví dụ chỉ hiện lựa chọn «Đòi nợ» nếu `debt > 0`.

Đối với học sinh THPT làm game jam hoặc đồ án Capstone, bài tập thực tế: visual novel 3 nhân vật, 2 ending — vẽ flow trước khi viết full text để tránh «viết xong mới phát hiện nhánh chết». Hub node gom nhiều flow vào một điểm (ví dụ quay về thị trấn) mà không copy-paste hàng chục lần.

  • Dialogue lines — speaker, emotion tag, voice-over note cho recording
  • Player choices — 2–4 option, mỗi option dẫn flow riêng
  • Jump & Hub — nhảy giữa fragment, tránh duplicate node

2. Entity, nhân vật & bible cốt truyện

Articy không chỉ lưu câu thoại — nó có database entity cho Character, Location, Item, Faction. Mỗi entity có template field (tuổi, mối quan hệ, concept art link) giúp writer và artist cùng tham chiếu. Khi gán speaker trong dialogue, bạn chọn từ database — đổi tên hiển thị một chỗ, cập nhật mọi flow.

Đây là bước «professionalize» so với viết truyện trong Word: game có 50 NPC vẫn quản lý được; localization team export danh sách string có ID ổn định. Học sinh có thể dùng phần này làm story bible cho dự án học đường — dù chưa export sang engine.

3. Export sang engine & làm việc nhóm

Articy có plugin Unity và Unreal — export project thành dữ liệu engine đọc được, giảm lỗi copy-paste thủ công. Programmer triển khai dialogue runner; writer tiếp tục sửa trong Articy và re-export. Bản Articy hỗ trợ multi-user (tùy license) và comment trên node — phù hợp team nhỏ remote.

Luồng indie điển hình: viết flow trong Articy → export → Unity Dialogue System hoặc custom JSON parser → playtest → sửa node trong Articy. Hiểu vòng lặp này giúp bạn thấy narrative design là công việc lặp iteration, không phải «viết xong một lần là xong».

03So sánh với phần mềm tương tự

Hai công cụ hay được so với Articy là Twine (miễn phí, hypertext cho interactive fiction) và Yarn Spinner / Ink (script dialogue gắn trực tiếp engine). Articy nghiêng về visual planning + database; Twine và Yarn/Ink nghiêng về viết nhanh và code-first — tùy quy mô game và vai trò bạn muốn đảm nhận.

Articy Draft vs Twine

Twine xuất bản web/HTML interactive fiction cực nhanh — không cần engine 3D, phù hợp học sinh THPT thử viết truyện rẽ nhánh và publish link ngay. Articy trả phí, hướng tới game commercial với export Unity/Unreal và entity management phức tạp hơn Twine passage đơn giản.

Nếu dự án là visual novel web hoặc text adventure học đường, Twine đủ dùng và miễn phí. Khi team chuyển sang Unity 2D/3D với hàng trăm node và biến trạng thái, Articy (hoặc tool tương đương) giúp scale mà không mất overview trên một canvas khổng lồ.

Điểm mạnh và yếu so sánh

Articy mạnh hơn: database nhân vật/item, export Unity/Unreal chính thức, condition phức tạp, workflow studio game.
Twine mạnh hơn: miễn phí, học trong vài giờ, publish web ngay, lý tưởng IF và đồ án văn tương tác.
Chọn Articy khi: làm game Unity/Unreal có dialogue system, team cần bible và export có cấu trúc.
Chọn Twine khi: text/hypertext fiction, game jam nhỏ, ngân sách zero và không cần engine 3D.

Articy Draft vs Yarn Spinner / Ink

Yarn Spinner (Unity) và Ink (Inkle, dùng rộng với Unity plugin) là ngôn ngữ script viết dialogue dạng text có cú pháp — programmer-friendly, version control Git dễ diff. Articy là GUI flowchart — writer-friendly, nhìn toàn cảnh nhánh trên canvas; export ra data engine thay vì viết script trực tiếp.

Nhiều studio indie dùng Ink/Yarn vì miễn phí và gắn chặt code; studio có budget narrative thường cân Articy cho writer không code. Học sinh biết cả hai: Ink cho prototype nhanh trong Unity; Articy cho portfolio «narrative designer» chuyên nghiệp hơn.

04Học Articy Draft như thế nào?

1
Làm quen canvas và node cơ bản

Tải trial Articy hoặc xem video chính thức; tạo project mới, flow fragment đầu tiên. Tuần đầu: thêm 5–7 dialogue node nối tuần tự, gán speaker tạm, chạy preview trong Articy. Mục tiêu là quen kéo-thả, zoom canvas và navigator — tương tự học sơ đồ tư duy nhưng mỗi node mang logic game.

2
Player choice và hai ending

Thêm node lựa chọn với 2 option dẫn hai nhánh; mỗi nhánh kết thúc bằng ending khác nhau. Viết ngắn gọn (visual novel demo 10 phút chơi). Bài học: plan ending trước khi viết chi tiết — tránh nhánh «viết hay» nhưng không khép được câu chuyện.

3
Entity database và template nhân vật

Tạo 3 character entity với field mô tả, quan hệ, avatar note; gán speaker từ DB thay vì gõ tên tay. Thêm Location và Item tham chiếu trong narration. Thói quen này mirror workflow studio — và giúp giáo viên/co-founder đọc bible mà không mở từng node.

4
Biến, condition và hub

Định nghĩa biến `friendship`, tăng giá trị sau lựa chọn «Giúp đỡ»; dùng Condition để mở hội thoại bí mật khi `friendship >= 3`. Thêm Hub «Quay về làng» gom nhiều nhánh. Đây là bước chuyển từ «truyện rẽ nhánh» sang «game state» — nền tảng cho quest design sau này.

5
Export thử sang Unity hoặc mock runtime

Nếu có Unity, cài Articy plugin và export project demo; nếu chưa, export JSON và đọc cấu trúc file để hiểu pipeline. Playtest với bạn cùng lớp, ghi bug «nhánh không vào được» — sửa flow và re-export. Iteration vòng này quan trọng hơn viết thêm 50 node chưa test.

6
Portfolio narrative design

Đóng gói một vertical slice: flowchart screenshot (che spoiler nếu cần), PDF story bible 3 nhân vật, link build Unity/Web nếu có. Ghi rõ vai trò (writer, narrative designer). So sánh thêm một dự án Twine hoặc Ink nhỏ để show đa kỹ năng. Tìm hiểu thêm khóa game writing, game jam (Global Game Jam Việt Nam) và vị trí junior narrative tại studio local — portfolio logic flow đôi khi thuyết phục hơn đoạn văn hay nhưng không có cấu trúc.
