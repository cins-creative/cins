Bạn vào trang GitHub của một game indie, thấy hàng trăm commit, nhánh feature và pull request được review — đó là cách team dev phân tán trên toàn cầu cùng làm một dự án mà không ghi đè code lẫn nhau. GitHub là nền tảng hosting Git lớn nhất thế giới: lưu mã nguồn, theo dõi lịch sử thay đổi, cộng tác qua pull request và tự động hóa build/test bằng GitHub Actions.

GitHub không phải ngôn ngữ lập trình — nó là công cụ quản lý phiên bản và cộng tác xoay quanh Git. Hiểu GitHub giúp học sinh THPT làm dự án nhóm (web, game, app), xây portfolio code công khai và chuẩn bị cho môi trường làm việc thực tế nơi «biết Git» thường là yêu cầu tối thiểu dù bạn frontend, backend hay game programmer.

01GitHub là gì?

GitHub do GitHub Inc. (thuộc Microsoft từ 2018) vận hành — nền tảng cloud cho repository Git, truy cập qua web, desktop app và CLI trên Windows, macOS, Linux. Git (phần mềm local) do Linus Torvalds tạo ra để quản lý mã nguồn Linux; GitHub thêm giao diện web, social coding và tích hợp CI/CD.

Khác với lưu file trên Google Drive hay gửi ZIP qua Zalo, GitHub lưu từng commit — snapshot có message mô tả thay đổi, ai sửa, khi nào. Branch cho phép làm tính năng mới song song; merge/pull request gộp code sau khi review. Giao diện có repo file tree, Issues, Pull Requests và Actions tab — ban đầu trông kỹ thuật, nhưng workflow cơ bản (clone → edit → commit → push) học được trong vài buổi.

  • Repository & version history — lưu code, docs, asset text; xem diff từng dòng thay đổi
  • Branch & pull request — làm việc nhóm, code review trước khi merge vào main
  • Issues & Projects — theo dõi bug, task và kanban board cho dự án nhóm
  • GitHub Actions & Pages — CI/CD tự động; host website tĩnh miễn phí từ repo

02Tính năng cốt lõi

1. Git cơ bản: commit, branch và remote

Nền tảng của GitHub là Git local trên máy bạn. Sau khi cài Git, bạn clone repo về, sửa file, dùng git add và git commit để lưu snapshot kèm message rõ ràng («fix player jump height», «add login form»). Branch (ví dụ feature/enemy-ai) tách dòng phát triển khỏi main; git push đẩy lên GitHub để người khác thấy.

Đối với học sinh THPT mới làm quen, thói quen tốt ngay từ đầu: commit nhỏ, message tiếng Việt hoặc Anh mô tả «why», không commit file .env chứa password — bài học bảo mật thực tế mà nhiều junior dev từng mắc phải.

  • clone / pull / push — đồng bộ code giữa máy local và remote GitHub
  • branch & merge — làm tính năng riêng, gộp khi xong và test
  • .gitignore — loại trừ node_modules, build folder khỏi repo

2. Pull request, code review và cộng tác nhóm

Pull Request (PR) là đề xuất merge branch kèm mô tả thay đổi, screenshot và link issue. Teammate comment từng dòng code, yêu cầu sửa trước khi approve — mô phỏng quy trình studio thật. Với dự án học sinh 3–5 người, PR tránh tình trạng «bạn A sửa xong bạn B ghi đè mất cả tuần».

Issues gán task («Implement shop UI», «Fix collision bug #12»); Projects board kéo thả cột Todo/In Progress/Done. README.md trong repo giải thích cách cài và chạy project — recruiter và giáo viên đọc đầu tiên khi mở link portfolio.

3. GitHub Actions, Pages và portfolio dev

GitHub Actions chạy workflow tự động khi push hoặc mở PR: chạy test, lint, build và deploy. Ví dụ push lên main → Actions build Next.js → deploy Vercel. GitHub Pages host site tĩnh miễn phí từ branch gh-pages — phù hợp portfolio HTML/CSS hoặc documentation project.

Profile README (repo trùng tên username) là «landing page» dev: badge, link project, stats commit. Học sinh THPT có 2–3 repo public sạch sẽ (game jam, web app trường) thường nổi bật hơn CV dài mà không có link code khi ứng tuyển internship IT.

03So sánh với phần mềm tương tự

Hai nền tảng hay được đặt cạnh GitHub nhất là GitLab (self-host, DevOps tích hợp) và Bitbucket (Atlassian, gắn Jira). Cả ba đều dùng Git — khác nhau ở ecosystem, giá và văn hóa cộng đồng open source.

GitHub vs GitLab

GitLab cung cấp DevOps all-in-one: Git repo, CI/CD built-in mạnh, container registry, security scan — nhiều doanh nghiệp self-host GitLab trên server riêng. GitHub thắng ở cộng đồng open source toàn cầu, Copilot tích hợp, Actions marketplace và là nơi recruiter tìm profile dev phổ biến nhất.

Điểm mạnh và yếu so sánh

GitHub mạnh hơn: ecosystem open source lớn nhất, portfolio visibility, tích hợp VS Code/Copilot, free tier hào phóng cho public repo.
GitLab mạnh hơn: CI/CD và DevOps tích hợp sâu, self-host miễn phí (Community Edition), phù hợp công ty cần kiểm soát data on-premise.
Chọn GitHub khi: học sinh, indie dev, open source và portfolio công khai — tiêu chuẩn de facto.
Chọn GitLab khi: công ty deploy GitLab nội bộ hoặc cần DevOps pipeline enterprise trên một nền tảng.

GitHub vs Bitbucket

Bitbucket (Atlassian) tích hợp chặt với Jira và Confluence — team agile đã dùng Atlassian stack thường chọn Bitbucket cho private repo. GitHub có Jira integration nhưng Bitbucket «native» hơn trong ecosystem đó. Bitbucket free cho team nhỏ private; GitHub free unlimited public + private repo cá nhân.

04Học GitHub như thế nào?

1
Tạo tài khoản và repo đầu tiên

Đăng ký github.com, tạo repo public «hello-portfolio» với README. Clone về máy (HTTPS hoặc SSH), mở bằng VS Code, sửa README, commit và push. Mục tiêu là hoàn thành vòng clone → edit → commit → push một lần — nhiều bạn THPT sợ terminal nhưng VS Code Source Control panel làm được hầu hết thao tác GUI.

2
Branch, merge và xử lý conflict cơ bản

Tạo branch add-feature, sửa file, merge vào main qua PR trên web (kể cả solo project để luyện thói quen). Cố ý tạo conflict đơn giản (hai branch sửa cùng dòng) và học git merge / resolve conflict. Thường 1–2 tuần sẽ tự tin hơn khi làm nhóm không panic khi Git báo conflict.

3
Dự án nhóm: Issues, PR review và quy ước

Với team 2–4 người (web trường, game jam), thống nhất: ai làm branch gì, ai review PR, dùng Issues gán task. Viết CONTRIBUTING.md ngắn: cách chạy project, naming branch (feature/..., fix/...). Mục tiêu là simulate studio nhỏ — communication quan trọng không kém code.

4
README chuyên nghiệp và GitHub Pages

Viết README có: mô tả dự án, screenshot/GIF, hướng dẫn install, tech stack, link demo. Bật GitHub Pages cho site tĩnh hoặc link Vercel trong README. Portfolio dev bắt đầu từ README tốt — recruiter spend 30 giây đầu ở đó.

5
GitHub Actions cơ bản và bảo mật

Thêm workflow YAML chạy npm test hoặc build khi push; học không commit secret — dùng GitHub Secrets cho API key. Kiểm tra Dependabot alert cho dependency lỗi bảo mật. Đây là bước nâng portfolio từ «có code» lên «có quy trình chuyên nghiệp».

6
Đóng góp open source và định hướng nghề

Fork repo nhỏ (docs fix, typo, translation), mở PR lên project open source — kinh nghiệm review thật. Pin 3–6 repo tốt nhất trên profile; thêm GitHub link vào CV/LinkedIn. Tìm hiểu GitHub Student Developer Pack (miễn phí tool cho học sinh). Dù sau này công ty dùng GitLab/Bitbucket, kỹ năng Git + workflow PR là transferable — đầu tư GitHub hôm nay trả lãi suốt sự nghiệp dev.
