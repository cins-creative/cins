Bạn mở điện thoại, vuốt qua màn hình đăng nhập app ngân hàng, bấm nút «Chuyển tiền» và thấy chuyển sang màn hình xác nhận với hiệu ứng chuyển cảnh mượt — trước khi lập trình viên viết một dòng code, team sản phẩm thường đã dựng prototype tương tự trong công cụ UI/UX. Adobe XD từng là lựa chọn phổ biến cho bước đó: thiết kế giao diện app và web, nối màn hình thành luồng tương tác, chia sẻ link cho stakeholder bấm thử.

XD thuộc hệ Adobe Creative Cloud, quen thuộc với designer đã dùng Photoshop hay Illustrator. Tuy Adobe đã ngừng phát triển tính năng mới cho XD (chuyển hướng sang Figma trong hệ sinh thái), phần mềm vẫn còn được dùng tại nhiều studio Việt Nam — và hiểu XD giúp bạn đọc được portfolio UI cũ, cũng như nắm nền tảng trước khi học Figma hoặc công cụ khác.

01Adobe XD là gì?

Adobe XD (Experience Design) là phần mềm thiết kế giao diện và prototype chạy trên Windows và macOS — tập trung vào UI cho app mobile, web và một số màn hình kích thước tùy chỉnh. Giao diện gọn hơn Photoshop: artboard ở giữa, panel layer bên trái, thuộc tính bên phải, chế độ Prototype để nối màn hình.

Khác với Illustrator (vector in ấn) hay Photoshop (chỉnh ảnh), XD được xây dự cho digital product design: component tái sử dụng, responsive resize cơ bản, design system nhẹ và xuất spec cho developer. File .xd lưu layout, style và link prototype trong một project — phù hợp học sinh THPT làm mockup app ý tưởng, báo cáo môn Tin học hoặc dự án khởi nghiệp số.

  • Artboard & layout — khung iPhone, Android, desktop, tablet có sẵn preset kích thước
  • Component & states — nút, icon, card tái sử dụng; Main Component đồng bộ bản sao
  • Prototype — hotspot, transition, overlay, voice prototype cho demo tương tác
  • Chia sẻ & handoff — link preview trên web, comment, export asset và spec CSS cho dev

02Tính năng cốt lõi

1. Thiết kế giao diện với vector và layout

XD cung cấp bộ công cụ vector cơ bản: hình chữ nhật, elip, đường path, Boolean operations — đủ để dựng wireframe grayscale và mockup full color mà không cần mở Illustrator. Repeat Grid cho phép nhân bản danh sách (feed tin tức, gallery ảnh) và chỉnh spacing hàng loạt — tiết kiệm thời gian so với copy-paste thủ công.

Responsive Resize giúp một số nhóm object co giãn khi đổi kích thước artboard — hữu ích khi thiết kế cùng layout cho mobile và tablet. Với học sinh THPT, bài tập thực tế: dựng 5–8 màn hình app «đặt đồ ăn trường học» hoặc «thư viện sách lớp» với typography và màu thống nhất.

  • Artboard preset — iPhone 15, Pixel, iPad, web 1920px sẵn có
  • Repeat Grid — lặp card/list và chỉnh khoảng cách đồng loạt
  • Layout grid & guides — căn lề 8px, 12-column grid cho web

2. Component, design system và states

Components trong XD hoạt động tương tự concept «master/instance»: tạo một nút Primary, mọi bản copy liên kết — đổi màu hoặc label ở Main Component thì instance cập nhật theo. Component States (hover, pressed, disabled) giúp mô phỏng trạng thái nút trong prototype mà không cần duplicate artboard.

Creative Cloud Libraries đồng bộ màu, character style và component giữa XD, Photoshop, Illustrator — team agency Adobe dùng chung palette thương hiệu. Học sinh có thể tạo mini design system: 3 màu brand, 2 font, bộ button/input — rèn tư duy hệ thống thay vì thiết kế từng màn hình rời rạc.

3. Prototype, chia sẻ và handoff developer

Chuyển sang tab Prototype, kéo dây nối từ object (nút, icon) sang artboard đích, chọn transition (dissolve, slide left, push) và thời gian. Overlay mô phỏng modal, menu dropdown; voice prototype (tính năng từng có) hỗ trợ demo flow bằng lệnh thoại trong một số phiên bản. Bấm nút Play để xem trước; publish lên Adobe cloud để gửi link — người xem không cần cài XD.

Handoff: developer mở link, xem khoảng cách pixel, copy mã CSS cơ bản, tải PNG/SVG asset. XD không mạnh bằng Figma Dev Mode ngày nay, nhưng đủ cho dự án học tập và team nhỏ. Luồng điển hình: design → prototype link → feedback comment → iteration → export asset sang React Native hoặc Flutter (dev tự implement).

03So sánh với phần mềm tương tự

Hai công cụ hay được đặt cạnh Adobe XD nhất là Figma (collaboration trên web, tiêu chuẩn ngành hiện tại) và Sketch (UI design trên macOS, phổ biến trước khi Figma lên ngôi). Không có câu trả lời duy nhất — tùy hệ sinh thái công ty, nền tảng máy và mục tiêu nghề nghiệp.

Adobe XD vs Figma

Figma chạy trên trình duyệt và desktop, mạnh về collaboration real-time: nhiều designer chỉnh cùng file, comment, version history và Dev Mode cho developer. Cộng đồng plugin, UI kit và tuyển dụng UI/UX tại Việt Nam ngày càng nghiêng về Figma — Adobe đã mua Figma (thương vụ bị regulatory chặn một phần nhưng quan hệ hợp tác vẫn tiếp diễn) và khuyến khích người dùng mới chọn Figma cho dự án greenfield.

XD vẫn có lợi thế nếu bạn đã trả Creative Cloud và làm việc chặt với Photoshop/Illustrator — Libraries đồng bộ màu và asset. Performance local app ổn trên máy cấu hình trung bình; Figma phụ thuộc mạng khi dùng bản web. Với học sinh THPT bắt đầu năm 2025–2026, lộ trình thực tế: học nguyên lý UI trên XD hoặc Figma miễn phí, ưu tiên Figma nếu mục tiêu thực tập startup/agency.

Điểm mạnh và yếu so sánh

Adobe XD mạnh hơn: liền mạch Creative Cloud; quen giao diện Adobe; đủ cho prototype và handoff cơ bản; Starter plan miễn phí giới hạn.
Figma mạnh hơn: collaboration đa người; ecosystem plugin/UI kit khổng lồ; Dev Mode; auto layout nâng cao; xu hướng tuyển dụng và dự án mới.
Chọn Adobe XD khi: team đã đầu tư Adobe stack, duy trì file .xd cũ, hoặc học trong môi trường chỉ có CC.
Chọn Figma khi: mục tiêu UI/UX nghề nghiệp, làm việc nhóm online, hoặc dự án mới cần scale collaboration.

Adobe XD vs Sketch

Sketch là phần mềm UI design chỉ trên macOS, từng thống trị cộng đồng designer Apple-centric trước Figma. Sketch mạnh về vector editing tinh chỉnh, symbol system và plugin phong phú trên nền Mac. File .sketch lưu local hoặc trên Sketch Cloud — collaboration đã cải thiện nhưng vẫn không bằng Figma native multiplayer.

So với XD: Sketch không có trên Windows — học sinh THPT dùng PC tại Việt Nam thường gặp rào cản. XD chạy cross-platform (Win/Mac). Sketch có cộng đồng plugin lâu đời cho iOS design; XD prototype tích hợp sẵn không cần plugin. Nhiều studio Việt Nam chuyển từ Sketch sang Figma; XD và Sketch đều là «thế hệ trước» so với Figma nhưng vẫn xuất hiện trong job description legacy.

04Học Adobe XD như thế nào?

1
Làm quen artboard và công cụ cơ bản

Tuần đầu, tạo file mới, chọn preset iPhone, vẽ rectangle, text, elip — luyện căn lề bằng smart guides. Tạo 3 artboard: Trang chủ, Chi tiết, Cài đặt. Mục tiêu: quen canvas vector và layer panel trước khi học component — nhiều bạn THPT nhảy vào prototype sớm mà chưa nắm spacing và typography cơ bản.

2
Wireframe grayscale trước, màu sau

Dựng luồng 5 màn hình app ý tưởng (ví dụ «quản lý CLB trường») chỉ dùng xám, placeholder text và icon đơn giản. Tập trung bố cục: header, nội dung, navigation bar. Sau khi luồng logic ổn, thêm màu brand, font (Adobe Fonts), ảnh từ Photoshop. Cách này giống quy trình UX thực tế: structure trước, visual sau.

3
Component và Repeat Grid

Tạo Main Component cho nút, thẻ tin, avatar — dùng instance trên các artboard. Dùng Repeat Grid cho danh sách bài viết hoặc sản phẩm. Thêm Component State cho hover (mô phỏng desktop) hoặc trạng thái disabled. Mục tiêu: sửa một chỗ, cập nhật toàn project — nền tảng cho design system sau này trên Figma cũng tương tự.

4
Prototype và chia sẻ link

Nối nút «Đăng nhập» sang màn Home, menu sang trang con, thêm transition slide. Publish prototype, gửi link cho bạn cùng lớp bấm thử trên điện thoại. Thu feedback: chỗ nào confusing, thiếu nút Back. Iteration 2–3 vòng — kỹ năng «test nhanh ý tưởng» mà startup và agency đều cần.

5
So sánh song song với Figma (1 tuần)

Dựng lại cùng một màn hình trên Figma (free tier). Quan sát khác biệt: auto layout, comment, multiplayer. Viết bảng so sánh ngắn — giúp bạn biện luận trong phỏng vấn thực tập «đã dùng XD, sẵn sàng chuyển Figma». Nếu trường dạy Figma, nguyên lý từ XD chuyển sang gần như trực tiếp.

6
Portfolio và định hướng UI/UX

Đăng 1 case study: vấn đề → wireframe → mockup → link prototype → bài học rút ra. Behance, Dribbble hoặc Notion cá nhân. Ghi rõ công cụ (XD + Photoshop nếu có). Tìm hiểu thêm UX research cơ bản (persona, user flow). Lộ trình dài hạn: nắm Figma, học HTML/CSS cơ bản để nói chuyện với developer — junior UI designer tại Việt Nam thường cần portfolio 2–3 project hơn chứng chỉ.
