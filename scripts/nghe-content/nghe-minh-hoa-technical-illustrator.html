Khi instruction manual IKEA có exploded view rõ ràng giúp DIY assembly, khi Apple's patent drawing precise + tell engineering story, khi Boeing 737 maintenance manual có technical diagram cho mechanic — đó là tác phẩm của Technical Illustrator.

Technical Illustrator là chuyên gia vẽ illustration kỹ thuật precise — engineering diagram, exploded view, instructional manual, patent drawing, scientific diagram. Khác commercial illustrator — focus accuracy + clarity over aesthetic. Critical cho engineering, manufacturing, patent, education, science publication.

01Technical Illustrator là ai?

Technical Illustrator là chuyên gia precision illustration cho technical document. Phân loại theo industry: (1) Engineering + manufacturing — automotive (Toyota, Ford), aerospace (Boeing, Airbus), heavy machinery (Caterpillar); (2) Consumer product — IKEA instruction, Apple manual, appliance manual; (3) Medical + healthcare — medical device, surgical instruction; (4) Patent — patent office submission requirement; (5) Scientific publication — research paper, textbook; (6) Military + defense — equipment manual; (7) Architecture + construction — building system diagram. Phân loại theo style: (1) Exploded view — assembly + part relationship; (2) Cutaway view — internal structure exposed; (3) Cross-section; (4) Schematic — circuit, plumbing; (5) Isometric / axonometric; (6) Perspective rendering.

Khác với Editorial Illustrator (aesthetic + emotion) — Technical accuracy primary. Khác với Scientific Illustrator — overlap, Scientific = biological / natural specific, Technical = engineering / mechanical. Khác với Engineering Drafter (CAD drawing) — Technical Illustrator more visual + presentation. Khác với 3D Visualizer — Technical can be 2D + 3D, lean toward 2D. Hợp tác chặt với: Engineer (mechanical, electrical, civil), Industrial Designer, Technical Writer (documentation), Patent attorney, Scientist + researcher, Publisher + production team.

02Công việc của một Technical Illustrator

1. Brief + source + reference

Brief receive: từ engineer / writer / patent attorney — what to illustrate, purpose, audience (engineer vs DIY user), spec. Source gather: engineering CAD file (SolidWorks, AutoCAD), photo of product, sketch, specification document. Reference research: similar illustration in industry, style guide (military STD, brand). Approach decide: 2D vs 3D, exploded vs cutaway, level of detail.

  • Brief + purpose
  • Source CAD + photo
  • Approach decide

2. Construct + draw — primary work

Core. 3D model build / receive: Adobe Illustrator có 3D capability, SolidWorks, Rhino, KeyShot. Angle + projection choose: isometric standard (no foreshortening), 3-quarter view, top-down. Exploded view build: part separate along axis, line indicate assembly. Cutaway draw: section cut + interior expose. Line art: clean vector line, hierarchy weight (thick outer contour, thin internal detail). Annotation + callout: label, leader line, part number, dimension.

03Technical Illustrator cần giỏi gì?

📐 Engineering drawing literacy

🔍 Precision + accuracy
⚙️ Technical literacy domain
📋 Standard + convention

Engineering drawing literacy

Read engineering drawing — plan, elevation, section, isometric. Dimension + tolerance. Threading + fastener convention. Read: "Engineering Drawing and Design" (Madsen), "Manual of Engineering Drawing" (Simmons).

Illustrator + CAD + 3D

Adobe Illustrator primary 2D. SolidWorks / Rhino / Fusion 360 3D source. KeyShot render. 3DS Max + Maya some studio. InDesign for manual layout. Adobe Dimension 3D + Illustrator hybrid.

Precision + accuracy

Every dimension precise. Every connection correct. No artistic license. Engineering audience scrutinize. Mistake = product fail / patent reject.

Technical literacy domain

Hiểu domain — mechanical, electrical, plumbing, fluid, fabrication. Read engineering textbook. Visit factory. Some Technical Illustrator have engineering degree.

Standard + convention

ATA Specification 2200 (aerospace), MIL-STD (military), ISO drawing standard. Patent office requirement (USPTO, EPO). Brand style guide. Industry convention.

Engineer communication

Translate engineer language to visual. Patient question (engineer assume knowledge). Diplomatic feedback (engineer ego protect). Write technical caption.
