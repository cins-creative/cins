02Công việc của một XR / AR Designer

1. Concept + research + spatial design

Foundation. Use case define: what problem solve, why XR (vs phone, vs PC). User research: target user, comfort, ability. Reference research: existing XR experience study. Spatial sketch: think in 3D space — diagram, paper sketch. Storyboard XR: scene + interaction beat. Hardware constraint research: FOV, tracking, comfort. Accessibility consideration: seated vs standing, hand vs gaze.

  • Use case + research

03XR / AR Designer cần giỏi gì?

Spatial UX + 3D thinking

Think in 3D + spatial. Window vs scene. Distance + scale. User comfort range. Read: "Designing VR" (Casey, Gardner-Stephen, Skillern), "Designing for Mixed Reality" (Cawood), Apple Human Interface Guidelines for visionOS.

Unity + Unreal

Unity dominant in mobile + Meta Quest. Unreal high-end + Apple Vision Pro. C# (Unity) + Blueprint (Unreal) scripting. Reality Composer Pro for visionOS. Spark AR for Instagram filter.

Interaction design (hand, gaze, voice)

Hand tracking + gesture. Eye tracking + gaze. Voice. Controller alternative. Read: "The VR Book" (Jerald), "Designing Virtual Reality Experiences" (Bucher).

3D modeling + scene

Blender free + powerful. Maya industry. Cinema 4D. Polygon optimize for XR FPS. UV + texture.

Spatial audio

3D positional audio critical for immersion. Wwise, FMOD. HRTF (Head-Related Transfer Function). Apple Spatial Audio + Dolby Atmos integrate.

Hardware + platform literacy

Apple Vision Pro, Meta Quest 3 + Quest Pro, HTC Vive, Valve Index, Pico, HoloLens 2. ARKit + ARCore. WebXR. Constant spec evolve.
