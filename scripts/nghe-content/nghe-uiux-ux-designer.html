Khi book vé bay Vietjet với 4 step minimum vs cũ 12 step, khi Tiki checkout có guest checkout không bắt đăng ký, khi Zalo Pay onboarding new user 3 step thấy demo trước rồi register, khi VinFast charging app có nearest station + plug type filter trực giác — đó là tác phẩm của UX Designer.

UX Designer là chuyên gia thiết kế flow + structure + interaction của digital product — wireframe, prototype, user journey, information architecture. Focus problem-solving + usability over visual. Role nền tảng của product design discipline. Without UX, product = beautiful nhưng confusing. UX = thinking before pixel-pushing.

01UX Designer là ai?

UX Designer là chuyên gia user experience design. Phân loại theo focus: (1) Interaction Designer — flow, behavior, micro-interaction; (2) Information Architect — IA, navigation, taxonomy; (3) UX Researcher (overlap); (4) Product Designer — UX + UI combined (common at startup); (5) Service Designer — broader service touchpoint. Phân loại theo career: (1) Junior UX Designer; (2) UX Designer; (3) Senior UX Designer; (4) Lead / Principal UX Designer; (5) Director of UX / VP of Design. Phân loại theo product: (1) Consumer app — MoMo, Tiki, Shopee; (2) Enterprise / SaaS — CRM, ERP, internal tool; (3) Fintech / banking; (4) Healthcare / public service; (5) Hardware-integrated — car infotainment, smart home. Phân loại theo employer: (1) Tech in-house; (2) Design agency; (3) Freelance; (4) Consultancy — McKinsey Digital, BCG Digital.

Khác với UI Designer — UI visual + aesthetic + polish, UX problem + flow + IA. Common conflated, dedicated UX deeper thinking. Khác với Product Manager — PM business + roadmap, UX user + experience. Tight collaboration. Khác với User Researcher — UXR research dedicated, UX Designer use research + design. Khác với Product Designer — Product = UX + UI combo. Hợp tác chặt với: UI Designer, Product Manager, UX Researcher, Frontend Developer, Engineer (technical feasibility), Content Designer / UX Writer, Service Designer.

Foundation. Stakeholder interview: PM + business — what problem, what goal. User research absorb: existing UXR insight + commission new research. Competitive analysis: how other product solve. Analytics review: where user drop, where struggle. Persona + journey synthesize: who, what need. Problem statement frame: clear, well-defined.

Systems + critical thinking

See big picture + edge case. Question assumption. First principle. Read: "The Design of Everyday Things" (Norman), "About Face" (Cooper), "Don't Make Me Think" (Krug).

IA + flow + wireframe

Information Architecture principle. Card sorting. Tree test. Navigation pattern. User flow diagram. Read: "Information Architecture for the World Wide Web" (Rosenfeld + Morville), "How to Make Sense of Any Mess" (Covert).

Figma + prototype tool

Figma industry standard. Sketch Mac legacy. Miro / FigJam workshop. Protopie / Framer advanced prototype. Maze / UserTesting remote usability test.

Research literacy

Conduct light research (interview, usability test). Hiểu research method. Collaborate với dedicated UXR. Read: "Just Enough Research" (Hall), "Lean UX" (Gothelf + Seiden).

Communication + critique

Articulate design rationale. Take + give critique. Stakeholder present. Long-term relationship build. Read: "Articulating Design Decisions" (Greever).

Data + analytics literacy

Google Analytics, Mixpanel, Amplitude. Funnel + cohort. A/B test fundamental. Read: "Lean Analytics" (Croll + Yoskovitz).
